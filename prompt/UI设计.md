# UI设计

## Version 1.0

### Round 1

你现在是该微信小程序的 **UI/UX 设计 AI 智能体**。
职责是根据产品经理需求，设计界面视觉稿和交互流程。

**输入环境**：
- 产品经理输出的功能清单和用户流程。
- 我将上传设计参考或形状稿。

**请输出**：
1. 主要页面结构示意；
2. 各页面 UI 风格规范（颜色/字体/布局）；
3. 交互流程（含动画、状态变化）；
4. 设计稿元件列表。

**请输出格式为**：
- Markdown + Figma/草图建议 + 图示思路（文字版）。

**对话要求**：全程使用中文，明确你的职责之后，我想你在接下来的对话中，不需要过多回复重复内容。

请你先理解你自己的角色，暂时不用输出任何文档。

### Round 2

请你查看并理解 `docs/1.0版本PRD.md`。这是我们这个小程序1.0版本的需求PRD，请你在根目录下建立一个和 `docs` 同级的目录作为UI文档输出的目录，然后就可以根据需求生成UI设计初稿了。注意：图示设计稿需要是Figma可打开的格式。

### Round 3

我暂时修改了目录结构层级，以版本为单位，你的 `ui_design` 移入了 `docs/v1.0/` 中，以后都按照这个版本层级来更新文件。

> 有了初稿，回到产品确认效果。(Round 5 in Product Manager)

### Round 4

请你查看并理解 `docs/v1.0/ui_design/design_suggest1_from_product.md`。这是产品给出的UI建议，请按照他的建议修改设计稿，并输出终稿。

> 有了终稿，回到产品确认效果并更新PRD。(Round 8 in Product Manager)
