# 宝可梦图鉴小程序功能验收指南 (Walkthrough)

## 1. 核心功能验收

### 1.1 首页 (Home) - 列表与搜索
- **路径**: `pages/home/index`
- **操作**:
    1. 打开小程序，应看到宝可梦卡片列表（Mock 数据展示 7 只）。
    2. 验证图片加载和属性徽章颜色是否正确。
    3. 点击顶部搜索栏，输入 `皮卡丘` 或 `25`，点击键盘“搜索”或失焦。
    4. **预期**: 列表应通过 Mock 逻辑过滤显示对应宝可梦；点击右侧“筛选”图标目前的提示为“开发中”。

### 1.2 详情页 (Detail) - 形态展示
- **路径**: `pages/detail/index`
- **操作**:
    1. 点击首页任意卡片（如“妙蛙种子”）。
    2. 进入详情页，查看大图、编号、属性、世代信息。
    3. 点击右下角 **✨闪光** 悬浮按钮。
    4. **预期**: 图片切换为闪光形态（颜色变化），按钮变为激活态（黄色）。
    5. 向下滑动查看“图鉴介绍”文本。

### 1.3 认证流程 (Auth) - 登录与注册
- **路径**: `pages/auth/index`
- **操作**:
    1. 进入“我的” Tab (`pages/profile/index`)，点击“登录/注册”卡片。
    2. 跳转至登录页，点击下方的“去注册”，标题应变更为“创建新账号”。
    3. 切换回登录模式，输入任意用户名和密码（非空），点击“登录”。
    4. **预期**: 弹出“登录成功”提示，1.5秒后自动返回个人中心。

### 1.4 个人中心 (Profile) - 状态变化
- **路径**: `pages/profile/index`
- **操作**:
    1. 登录成功后返回。
    2. **预期**: 顶部展示刚才输入的用户名和 Mock 头像 (皮卡丘)。
    3. 底部出现“退出登录”按钮。
    4. 点击“退出登录”，确认弹窗。
    5. **预期**: 状态重置为“未登录”，显示游客图标和引导文案。

## 2. 目录结构说明

```
miniprogram/
├── components/          # 公共组件
│   ├── pokemon-card/    # 列表卡片
│   ├── type-badge/      # 属性徽章
│   └── search-bar/      # 搜索栏
├── constants/           # 常量定义 (Types, Colors)
├── models/              # TS 接口定义 & Mock 数据
├── pages/               # 页面视图 (Home, Detail, Profile, Auth)
├── utils/               # 工具函数
├── app.ts               # 全局逻辑
└── app.scss             # 全局样式变量
```

## 3. 下一步建议

当前版本使用 Mock 数据实现了纯前端闭环。接下来的工作重点是：
1. **对接后端接口**: 替换 `models/mock-data.ts` 中的本地数据，在 services 层封装 `wx.request`。
2. **完善筛选功能**: 实现首页的 `Filter Sheet` 组件。
3. **性能优化**: 针对长列表引入虚拟滚动 (Recycle View) - 如果数据增加到 1000+。
