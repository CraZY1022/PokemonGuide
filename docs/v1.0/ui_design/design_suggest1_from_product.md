# UI 设计初稿审核反馈 (V1.0 MVP 版)

基于 V1.0 快速上线的目标，建议在以下几个“高耗时”但“非核心”的视觉/交互点上做减法，以加速开发：

### 1. 简化“动态背景色”
*   **原设计**：详情页头部背景根据宝可梦主属性变色（如火系淡红）。
*   **调整建议**：**V1 统一使用浅灰色或品牌淡红色背景**。
*   **理由**：18 种属性颜色的适配和与文字颜色的对比度调整（Accessibility）在开发调试上非常耗时。V1 先做统一，V1.1 再在此基础上增加属性色映射。

### 2. 简化“形态切换动画”
*   **原设计**：3D 翻转 (Flip) 或 交叉淡入淡出。
*   **调整建议**：**直接切换图片 (无动画或简单的 Opacity 过渡)**。
*   **理由**：小程序端的 3D 变换有时存在兼容性坑，且图片加载速度不一时动画效果会打折。优先保证“点一下能变”的功能响应速度。

### 3. 简化“字体引用”
*   **原设计**：数字使用 Roboto 或 DIN Condensed。
*   **调整建议**：**V1 直接使用系统默认无衬线字体 (System San-serif)**。
*   **理由**：引入自定义字体文件会增加包体积（小程序有 2MB 限制）和加载时间。除 Logo 外，尽量不引入第三方字体。

### 4. 筛选交互逻辑
*   **原设计**：点选后实时更新列表。
*   **调整建议**：**点选后需点击“确定”按钮触发筛选**。
*   **理由**：实时筛选对性能要求高（尤其在长列表下），且容易在用户连续点击时造成抖动。

**总结**：设计风格无需大改，建议在**实现细节**上降级处理，优先保证功能的稳定和流畅。
